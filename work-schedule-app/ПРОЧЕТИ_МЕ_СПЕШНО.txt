ПРОБЛЕМ:
--------
❌ Грешки при зареждане на Events страницата
❌ "Failed to load resource: 404"
❌ Има текст на украински "Події"

РЕШЕНИЕ В 3 СТЪПКИ:
--------------------

✅ СТЪПКА 1: Текстът е вече поправен на български!
   - "Події" → "Евенти" ✅
   - Refresh браузъра (F5)

✅ СТЪПКА 2: Изпълни SQL скрипта в Supabase
   
   1. Отвори: https://supabase.com/dashboard
   2. Влез в проекта си
   3. SQL Editor (лявото меню) → New Query
   4. Копирай ЦЕЛИЯ код от файла:
      
      📄 SIMPLE_EVENTS_SETUP.sql
      
   5. Постави го в SQL Editor
   6. Натисни "Run" (или F5)
   7. Чакай да завърши

✅ СТЪПКА 3: Тествай
   
   1. Затвори и отвори браузъра (или F5)
   2. Влез в приложението
   3. Отиди на Events
   4. Създай тестово събитие

═══════════════════════════════════════════════════════

ВАЖНО:
------
• Без SQL скрипта, Events НЕ ще работи!
• Таблиците events и votes ТРЯБВА да се създадат в Supabase
• След SQL скрипта, всичко ще работи перфектно!

СЪМНЕНИЯ?
---------
След SQL скрипта изпълни в Supabase SQL Editor:

    SELECT * FROM events;
    SELECT * FROM votes;

Ако има грешка "relation does not exist":
→ SQL скриптът НЕ е изпълнен! Опитай отново!

Ако няма грешка (дори празни таблици):
→ Всичко е ОК! ✅ Refresh браузъра!
